# Тестовое задание: Реализация формы регистрации

## Описание проекта

Этот проект представляет собой реализацию HTML-страницы с формой регистрации, которая включает в себя валидацию полей формы средствами JavaScript. Проект выполнен без использования сторонних JS-фреймворков и соответствует строгим требованиям по валидации и кроссбраузерной совместимости.

---

## Задание

### 1. Реализовать HTML-страницу с формой регистрации

Форма должна содержать следующие поля:

- **Имя** (`id="first-name"`)
- **Фамилия** (`id="last-name"`)
- **Email** (`id="email"`)
- **Пароль** (`id="password"`)
- **Подтверждение пароля** (`id="password-confirm"`)
- **Дата рождения** (`id="birth-day"`)

Также должна присутствовать кнопка отправки формы с `id="form-button"`. Все поля должны иметь соответствующие заголовки. Требований к расположению элементов на странице нет.

---

### 2. Валидация полей формы (JavaScript)

#### Требования к валидации:

1. **Имя и Фамилия**:
   - Содержат только допустимые символы.
   - Должны иметь адекватное ограничение по длине (обоснование длины приводится в комментариях к коду).

2. **Email**:
   - Должен содержать только валидный адрес электронной почты.

3. **Пароль и Подтверждение пароля**:
   - Значения полей должны совпадать.
   - Минимальная длина пароля — 8 символов.
   - Пароль должен содержать:
     - Минимум одну цифру.
     - Минимум одну заглавную букву.
     - Минимум одну строчную букву.
     - Минимум один специальный символ.

4. **Дата рождения**:
   - Возраст пользователя на момент регистрации должен быть не младше 18 лет.

5. **Общие требования к валидации**:
   - Валидация данных должна происходить после ввода пользователем и потери фокуса на элементе формы.
   - Кнопка "Отправить" (`id="form-button"`) должна быть заблокирована (`disabled`), пока пользователь не введёт валидные значения во все поля.
   - Пользователь должен получать понятные сообщения об ошибках, чтобы минимизировать количество попыток исправления.

---

### Требования к реализации

1. Использование любых JS-фреймворков категорически запрещено.
2. Использование примеров из интернета, реализующих части задания, запрещено.
3. Код должен корректно работать в последних версиях браузеров:
   - Firefox
   - Chrome
   - Edge
4. Типы элементов формы должны соответствовать вводимым значениям (например, `type="email"` для поля электронной почты).
5. HTML-код страницы должен проходить проверку через [W3C Validator](https://validator.w3.org/). Все предупреждения, ошибки и предложения по оптимизации должны быть устранены.
6. Каждый элемент формы должен иметь уникальный атрибут `id`, включая сообщения об ошибках.
7. Придерживаться следующих наименований для `id` элементов:
   - Поле имени: `id="first-name"`
   - Поле фамилии: `id="last-name"`
   - Поле email: `id="email"`
   - Поле пароля: `id="password"`
   - Поле подтверждения пароля: `id="password-confirm"`
   - Поле даты рождения: `id="birth-day"`
   - Кнопка отправки: `id="form-button"`

8. По результатам валидации:
   - Элементы формы (теги `<form>`, `<input>`, `<textarea>` и т.д.) должны получать класс:
     - `"invalid"` — если поле не прошло валидацию.
     - `"valid"` — если поле прошло валидацию.

9. Кнопка отправки должна разблокироваться сразу после валидного заполнения всех полей (без необходимости дополнительного клика вне формы).

10. Результат задания должен быть представлен в виде ZIP-архива, содержащего все необходимые файлы для работы проекта:
    - После распаковки архива проект должен открываться в браузерах (см. пункт 3) без ошибок.

---

## Как запустить проект

1. Распакуйте архив с проектом.
2. Откройте файл `index.html` в любом из поддерживаемых браузеров (Firefox, Chrome, Edge).
3. Заполните форму регистрации и проверьте работу валидации.
